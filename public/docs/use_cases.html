JB Tech Nepal
Use Case Documentation for Users and Administrators

1. Introduction

This document outlines the use cases for the JB Tech Nepal, a comprehensive online store that specializes in electronic products such as computers, laptops, mobile phones, tablets, and accessories. The platform enables users to browse products, compare specifications, make informed purchasing decisions, and complete transactions. The system is managed by administrators who oversee product listings, user accounts, orders, and system settings.

1.1 System Overview

The JB Tech Nepal is designed to provide customers with a convenient and informative shopping experience for electronic devices and accessories. It offers standard ecommerce functionality along with specialized features for electronics shopping, including detailed technical specifications, product comparisons, and AI-powered recommendations to help users find the best products for their needs and budget.

1.2 User Roles

The system recognizes two primary roles:

  • Regular Users: Customers who browse electronics products, view technical specifications, compare devices, place orders, and manage their accounts.
  • Administrators: Staff members who manage the website, including product listings, categories, customer information, orders, returns, and system settings.

2. User Use Cases

UC-U1: Account Management

Actor: Regular User

Description: Users can create accounts, log in, update their profile information, view their order history, and manage their contact information.

Flow:

  1. User registers for a new account by providing required information
  2. User receives confirmation email
  3. User logs in with their credentials
  4. User can view and edit their profile information
  5. User can change their password
  6. User can view their order history

UC-U2: Product Browsing

Actor: Regular User

Description: Users can browse electronic products by category (laptops, smartphones, tablets, accessories, etc.), use search functionality, filter results by specifications (processor, RAM, storage, screen size, etc.), and view detailed product information.

Flow:

  1. User navigates to product categories from the main menu (e.g., Computers, Phones, Audio)
  2. User can browse products within specific categories and subcategories (e.g., Gaming Laptops, Business Laptops)
  3. User can filter products based on technical specifications (RAM, processor, storage), price range, brand, etc.
  4. User can search for specific products using the search bar (e.g., "iPhone 13 Pro", "Dell XPS 15")
  5. User can view detailed product pages with technical specifications, images, and pricing
  6. User can view related products and accessories

UC-U3: Product Comparison

Actor: Regular User

Description: Users can select multiple electronic products to compare side by side, highlighting differences in technical specifications such as processor speed, RAM capacity, display quality, and battery life.

Flow:

  1. User selects products to compare by clicking "Add to Compare" on product cards (e.g., comparing different laptop models)
  2. User navigates to the comparison page
  3. System displays products side by side with their technical specifications
  4. System highlights key differences between products (e.g., better RAM in one laptop, faster processor in another)
  5. User can filter which specifications to compare (e.g., focus only on performance specs)
  6. User can remove products from comparison or add new ones
  7. User can request AI-powered recommendations based on comparison criteria

UC-U4: Shopping Cart Management

Actor: Regular User

Description: Users can add electronic products to their shopping cart, update quantities, remove items, and proceed to checkout.

Flow:

  1. User adds products to cart from product page or comparison page
  2. User views the cart contents
  3. User can update product quantities
  4. User can remove products from cart
  5. User can add recommended accessories for their selected devices
  6. User can continue shopping or proceed to checkout
  7. System calculates subtotal, tax, and shipping costs

UC-U5: Checkout Process

Actor: Regular User

Description: Users can complete the purchase by providing shipping information, selecting payment methods, and confirming the order.

Flow:

  1. User enters or selects shipping address
  2. User selects shipping method
  3. User is offered warranty or protection plans for electronic devices
  4. User selects payment method
  5. User reviews order summary
  6. User confirms the order
  7. System processes payment
  8. System displays order confirmation and sends confirmation email

UC-U6: Order Tracking

Actor: Regular User

Description: Users can track the status of their electronics orders, view order details, and track shipping information.

Flow:

  1. User navigates to the order history page
  2. User selects an order to view details
  3. User can see order status, items, quantities, prices, and shipping information
  4. User can track shipping if tracking information is available
  5. User can view warranty information for purchased devices
  6. User can print or download order invoice

UC-U7: Return Request

Actor: Regular User

Description: Users can initiate return requests for electronic products they have purchased, providing reason (defective, not as described, etc.) and condition information.

Flow:

  1. User navigates to order details
  2. User selects "Request Return" option
  3. User selects items to return
  4. User provides reason for return (e.g., defective screen, wrong model) and condition of items
  5. User uploads photos of defective items if applicable
  6. User submits return request
  7. System confirms return request and provides return instructions
  8. User can track return status

UC-U8: Technical Support

Actor: Regular User

Description: Users can contact technical support with questions about product specifications, compatibility issues, or troubleshooting for purchased electronics.

Flow:

  1. User navigates to the support page
  2. User views self-help resources like FAQs and troubleshooting guides
  3. User fills out technical support form with device details, issue description, and contact information
  4. User submits the form
  5. System confirms receipt of support request
  6. User receives auto-reply email with support ticket number
  7. User can view support ticket history in their account

UC-U9: AI-Powered Product Recommendation

Actor: Regular User

Description: Users can request and receive AI-generated recommendations for electronic products based on their preferences, usage requirements, and budget constraints.

Flow:

  1. User navigates to the product recommendation tool
  2. User specifies their requirements (e.g., laptop for gaming or business, budget range)
  3. User specifies must-have features (e.g., minimum RAM, dedicated graphics)
  4. System analyzes user preferences and available product data
  5. System presents personalized recommendations with explanations (e.g., "This laptop is recommended for its superior gaming performance")
  6. User can refine criteria for better recommendations
  7. User can add recommended products to comparison or cart

3. Administrator Use Cases

UC-A1: Dashboard Management

Actor: Administrator

Description: Administrators can access a dashboard providing an overview of system statistics, recent orders, customer activity, and alerts.

Flow:

  1. Admin logs into the administrative panel
  2. System displays dashboard with key metrics (sales, top-selling electronics, low stock alerts)
  3. Admin can view recent orders, new customers, and system alerts
  4. Admin can view performance metrics by product category (computers, phones, accessories)
  5. Admin can navigate to various management sections from the dashboard
  6. Admin can customize dashboard view and widgets

UC-A2: Product Management

Actor: Administrator

Description: Administrators can add, edit, and remove electronic products, including detailed technical specifications, images, pricing, and inventory.

Flow:

  1. Admin navigates to product management section
  2. Admin can view list of all products with filtering and search options
  3. Admin can create new products with detailed technical specifications (processor, RAM, storage, display, etc.)
  4. Admin can upload multiple product images showing different angles and features
  5. Admin can set product pricing, discounts, and availability
  6. Admin can manage product inventory and stock levels
  7. Admin can batch import products from manufacturer databases via API or CSV upload

UC-A3: Category Management

Actor: Administrator

Description: Administrators can create, edit, and organize electronic product categories and subcategories, including specification types for each category.

Flow:

  1. Admin navigates to category management section
  2. Admin can view category hierarchy (e.g., Electronics > Computers > Laptops > Gaming Laptops)
  3. Admin can create new categories and subcategories
  4. Admin can edit category details, including name, description, and images
  5. Admin can define specification types for each category (e.g., processor speed for computers, camera resolution for phones)
  6. Admin can organize the order of categories and specifications
  7. Admin can deactivate or delete categories (if not in use)

UC-A4: Order Management

Actor: Administrator

Description: Administrators can view, process, and manage customer orders for electronic products, including order status updates, payment verification, and shipping information.

Flow:

  1. Admin navigates to order management section
  2. Admin can view all orders with filtering options (by date, status, customer, etc.)
  3. Admin can view detailed order information including purchased electronic devices
  4. Admin can verify serial numbers for high-value electronics
  5. Admin can update order status (processing, shipped, delivered, etc.)
  6. Admin can update payment status
  7. Admin can add tracking information
  8. Admin can generate and print invoices
  9. Admin can communicate with customer about the order
  10. Admin can export order data

UC-A5: Return Management

Actor: Administrator

Description: Administrators can process return requests for electronic products, approve or reject returns, assess item condition, and process refunds.

Flow:

  1. Admin navigates to return management section
  2. Admin can view all return requests with status information
  3. Admin can review detailed return request with reason and item condition
  4. Admin can review photos of defective items submitted by customers
  5. Admin can approve or reject return requests
  6. Admin can update return status (approved, received, inspected, processed)
  7. Admin can process refunds, exchanges, or repairs for defective electronics
  8. Admin can add notes and communicate with customer
  9. Admin can update inventory when items are returned

UC-A6: Customer Management

Actor: Administrator

Description: Administrators can view and manage customer accounts, including account details, purchase history, and communication.

Flow:

  1. Admin navigates to customer management section
  2. Admin can view list of all customers with search and filtering options
  3. Admin can view detailed customer information and order history
  4. Admin can view customer's purchased devices and warranty information
  5. Admin can edit customer information
  6. Admin can reset customer passwords
  7. Admin can enable/disable customer accounts
  8. Admin can view and respond to customer messages
  9. Admin can export customer data

UC-A7: Inventory Management

Actor: Administrator

Description: Administrators can manage electronic product inventory, update stock levels, receive notifications for low stock, and view inventory history.

Flow:

  1. Admin navigates to inventory management section
  2. Admin can view current stock levels for all electronic products
  3. Admin can update stock quantities manually
  4. Admin can track serial numbers for high-value electronics
  5. Admin can adjust inventory (add or remove stock)
  6. Admin can set low stock thresholds for notifications
  7. Admin can view inventory history for each product
  8. Admin can export inventory data

UC-A8: Content Management

Actor: Administrator

Description: Administrators can manage website content, including promotional banners for new electronic devices, tech news, and deals.

Flow:

  1. Admin navigates to content management section
  2. Admin can create, edit, and publish web pages
  3. Admin can manage banners for new product launches (e.g., "New iPhone Release")
  4. Admin can create tech guides and tutorials for popular products
  5. Admin can setup and manage advertisements for featured electronics
  6. Admin can track advertisement performance
  7. Admin can schedule content publication

UC-A9: Settings Management

Actor: Administrator

Description: Administrators can configure various system settings, including general settings, SEO settings, contact information, and social media integration.

Flow:

  1. Admin navigates to settings management section
  2. Admin can configure general system settings
  3. Admin can manage SEO settings for better search engine visibility
  4. Admin can update contact information displayed on the website
  5. Admin can configure social media links and sharing options
  6. Admin can manage email templates for various notifications
  7. Admin can configure payment and shipping options

UC-A10: User Administration

Actor: Administrator

Description: Admin users can manage other administrators and staff accounts, including access permissions and roles.

Flow:

  1. Admin navigates to user administration section
  2. Admin can view list of all staff accounts
  3. Admin can create new staff accounts with specific roles
  4. Admin can assign specialized roles (e.g., technical support specialist, product manager)
  5. Admin can edit staff information and permissions
  6. Admin can disable or delete staff accounts
  7. Admin can assign roles with different levels of access
  8. Admin can view staff activity logs

6. Conclusion

This document has outlined the primary use cases for both regular users and administrators of the JB Tech Nepal. These use cases serve as a foundation for understanding the system's functionality and how different components interact with each other.

The JB Tech Nepal combines traditional ecommerce capabilities with specialized features for electronic products, providing a comprehensive platform for both customers and administrators. Users can browse electronic products, view detailed technical specifications, make informed comparisons, receive AI-powered recommendations, and complete purchases. Administrators have the tools needed to manage products, categories, orders, returns, inventory, customers, content, and system settings.

As the system evolves, additional use cases may be identified and existing ones may be refined to better meet the needs of all stakeholders. This living document should be updated regularly to reflect changes in system functionality and requirements.
